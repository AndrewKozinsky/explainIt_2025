# Перевод предложений из книг и фильмов

## Что делает функционал
В приложении пользователь может нажать на слово в предложении из книги или субтитров и получить:
- перевод всего предложения;
- перевод выделенной фразы с примерами.

Это помогает учить язык в контексте конкретного материала, а не отдельными словами вне сюжета.

## Серверный поток обработки
Клиент работает в два шага:
1. Сначала запрашивает уже существующие переводы через GraphQL query:
   - `getSentenceTranslation` — получить сохранённый перевод предложения по `sentenceId`;
   - `getPhraseTranslation` — получить сохранённый перевод фразы по `sentenceId` и диапазону offset.
2. Если нужного перевода нет, вызывает GraphQL мутации генерации:
   - `translateSentence` — сгенерировать новый перевод всего предложения;
   - `translatePhrase` — сгенерировать новый перевод выделенной фразы.

Основная цепочка на сервере:
`translate.resolver.ts` -> CQRS command handlers:
- `TranslateSentenceHandler`;
- `TranslatePhraseHandler`.

Входы:
- query для предложения принимает `sentenceId`;
- query для фразы принимает `sentenceId`, `selectedWordStartOffset`, `selectedWordEndOffset`;
- мутации принимают, в том числе, такие параметры:
  - `sentenceId` и `text` (обязательные);
  - `sourceLanguageCode`, `targetLanguageCode`;
  - контекст материала: `bookName`, `bookAuthor`, `videoName`, `videoYear`.

Query работают как read-only и возвращают `null`, если записи в БД нет.
Обе мутации работают синхронно и возвращают готовый JSON-результат.
Важно: мутации не делают проверку существующего перевода для возврата cached-результата — запрос на мутацию всегда идёт в путь генерации нового перевода.

Языки по умолчанию:
- `sourceLanguageCode` -> `en`, если не передан;
- `targetLanguageCode` -> `ru`, если не передан.

## LLM и формат результата
Сейчас провайдер по умолчанию — DeepSeek (в коде есть возможность переключения на ChatGPT).

Форматы ответов:
- для `translateSentence` ожидается строка перевода предложения;
- для `translatePhrase` ожидается JSON с полями `phrase`, `translate`, `examples`.

Сервер сохраняет результат в БД:
- `translation` — перевод предложения;
- отдельные переводы фраз — в таблице `SentencePhraseTranslation` (с диапазоном offset, статусом и примерами).

Для фразового перевода создаётся `pending`-запись, затем после ответа LLM запись обновляется в `ready` или `error`.

### Специфика фразового перевода
- Вход для фразы обязательно содержит `selectedWordStartOffset` и `selectedWordEndOffset`; сервер проверяет корректность диапазона.
- После ответа LLM сервер пытается привязать `phrase` к реальной подстроке предложения.
- Результат генерации сохраняется в новую `pending`-запись этого запроса (после генерации -> `ready` / при ошибке -> `error`).
- При ошибке генерации `pending`-запись помечается как `error`.

## Правила доступа
- Анонимный пользователь может работать только с публичными материалами, у которых включён `free_to_use`.
- Для публичных, но не `free_to_use` материалов анонимный доступ запрещён.
- Любая активная подписка даёт безлимитный доступ к переводу публичных материалов.
- Приватные материалы доступны только владельцу.
- Создание нового перевода для приватного материала разрешено только при стандартной подписке с положительным балансом.

Важно: для приватных материалов чтение уже существующего перевода и создание нового перевода проверяются отдельно.
- Владелец может читать существующий перевод даже без баланса.
- Но создание нового перевода без стандартной подписки с балансом запрещено.

## Лимит переводов
Для пользователей без подходящей подписки действует дневной лимит переводов.

Текущая настройка лимита задаётся в конфиге (сейчас: `3` в день).
Подсчёт хранится в Redis и ведётся по уникальным `sentenceId` в пределах суток.
Это значит, что повторный запрос того же предложения в тот же день не расходует лимит повторно.

Лимит применяется к операциям перевода по `sentenceId` (и при чтении, и при генерации, если доступ для пользователя определяется режимом `dailyLimit`).

## Списание баланса
- Списание токенов происходит только после успешной генерации LLM.
- Для `translateSentence` списание зависит от выбранного провайдера (`DeepSeek` или `ChatGPT`).
- Для `translatePhrase` используется списание DeepSeek-токенов.
