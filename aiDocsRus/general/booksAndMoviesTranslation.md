# Перевод предложений из книг и фильмов

## Что делает функционал
В приложении пользователь может нажать на слово в предложении из книги или субтитров и получить:
- перевод всего предложения;
- перевод выделенной фразы с примерами.

Это помогает учить язык в контексте конкретного материала, а не отдельными словами вне сюжета.

## Серверный поток обработки
Клиент вызывает GraphQL мутации:
- `translateSentence` — перевод всего предложения;
- `translatePhrase` — перевод выделенной фразы в предложении.

Основная цепочка на сервере:
`translate.resolver.ts` -> CQRS command handlers:
- `TranslateSentenceHandler`;
- `TranslatePhraseHandler`.

Мутации принимают, в том числе, такие параметры:
- `sentenceId` и `text` (обязательные);
- `sourceLanguageCode`, `targetLanguageCode`;
- контекст материала: `bookName`, `bookAuthor`, `videoName`, `videoYear`.

Обе мутации работают синхронно и возвращают готовый JSON-результат.

Если перевод для `sentenceId` уже существует в базе, сервер возвращает сохранённый результат сразу и не запрашивает LLM повторно.

## LLM и формат результата
Сейчас провайдер по умолчанию — DeepSeek (в коде есть возможность переключения на ChatGPT).

Форматы ответов:
- для `translateSentence` ожидается строка перевода предложения;
- для `translatePhrase` ожидается JSON с полями `phrase`, `translate`, `examples`.

Сервер сохраняет результат в БД:
- `translation` — перевод предложения;
- отдельные переводы фраз — в таблице `SentencePhraseTranslation` (с диапазоном offset, статусом и примерами).

Для фразового перевода создаётся `pending`-запись, затем после ответа LLM запись обновляется в `ready` или `error`.

## Правила доступа
- Анонимный пользователь может работать только с публичными материалами, у которых включён `free_to_use`.
- Для публичных, но не `free_to_use` материалов анонимный доступ запрещён.
- Любая активная подписка даёт безлимитный доступ к переводу публичных материалов.
- Приватные материалы доступны только владельцу.
- Создание нового перевода для приватного материала разрешено только при стандартной подписке с положительным балансом.

## Лимит переводов
Для пользователей без подходящей подписки действует дневной лимит переводов.

Текущая настройка лимита задаётся в конфиге (сейчас: `3` в день).
Подсчёт хранится в Redis и ведётся по уникальным `sentenceId` в пределах суток.
Это значит, что повторный запрос того же предложения в тот же день не расходует лимит повторно.
